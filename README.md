# foodie-radar

신뢰할 만한 맛집러, 공식 가이드, 추천 시그널, 광고 리스크, 개인 취향을 Markdown으로 관리하는 맛집 레이더입니다.

단순히 “근처 맛집 추천해줘”가 아니라, 다음 질문에 답하기 위한 구조입니다.

- 이 식당을 누가 직접 추천했나?
- 미슐랭, 빕 구르망, 블루리본 같은 공식 가이드에는 들어가 있나?
- 추천이 광고/협찬/초대 기반은 아닌가?
- 추천자가 평소 내 취향과 맞는가?
- “인생 맛집”, “국내 1위”, “장인의 퀄리티”, “자주 가는 단골” 같은 강한 추천 시그널이 있는가?
- 실제 방문 후보로 올릴 만큼 메뉴, 가격, 위치, 운영 정보가 충분한가?

## 핵심 아이디어

`foodie-radar`는 맛집 추천을 여러 층으로 나눠 관리합니다.

1. 공식 가이드 레이어
   - 미슐랭 스타/셀렉티드
   - 빕 구르망
   - 블루리본

2. 맛집러/인플루언서 레이어
   - 신뢰할 만한 유튜버, 인스타그램, 블로그, 지도 리스트를 관리합니다.
   - 단순 유명세가 아니라 추천 품질, 광고 리스크, 취향 적합성을 봅니다.

3. 식당 원장부 레이어
   - 식당명, 지점, 별칭, 위치, 장르, 메뉴, 가격, 출처를 표준화합니다.

4. 사용자 취향 레이어
   - 내가 좋아하는 추천 방식과 음식 취향을 기록합니다.
   - 예: 분야 최고 맛집, 극강의 가성비, 장인의 퀄리티, 인생 추천, 단골 맛집.

5. 방문 후보 큐
   - 지금 당장 갈 만한 곳, 보류할 곳, 운영 확인이 필요한 곳을 분리합니다.

## 왜 만들었나

일반 검색은 너무 얕습니다. 블로그 광고, 지도 평점, 바이럴 글만 보면 “왜 가야 하는지”가 잘 안 보입니다.

반대로 신뢰할 만한 맛집러 몇 명만 수동으로 따라가기도 한계가 있습니다. 영상 제목은 어그로성일 수 있고, Shorts는 식당명이 제목에 없을 수 있으며, 인스타그램이나 설명란에만 핵심 정보가 들어가기도 합니다.

그래서 이 프로젝트는 맛집 정보를 다음 방식으로 관리합니다.

- 신뢰할 만한 맛집러 리스트를 유지합니다.
- 맛집러별 추천 식당 목록을 매일 또는 주기적으로 갱신합니다.
- 광고/협찬/홍보성 콘텐츠는 별도 표시하거나 제거 후보로 둡니다.
- 내 취향과 맞는 추천자, 나보다 더 고급 취향을 가진 추천자를 구분합니다.
- 식당별로 3줄 추천 이유와 메뉴별 추천 사유를 기록합니다.
- 코스요리/오마카세는 주요 코스와 세부 메뉴 특징까지 확인합니다.

## 표준 추천 양식

```markdown
최신 확인 기준: YYYY-MM-DD
범위/조건:
주의: 운영상태, 휴무, 예약 가능 여부는 방문 전 재확인 필요.

## 가격대/조건 Top N

| 순위 | 식당 | 3줄 추천 | 메뉴별 추천 사유 | 가격대 | 위치 | 출처/주의 |
|---:|---|---|---|---|---|---|
| 1 | 식당명 | 1) 강한 추천 신호. <br>2) 내 취향과 맞는 이유. <br>3) 언제 가면 좋은지. | `메뉴 A`: 왜 대표 메뉴인지. <br>`메뉴 B`: 무엇을 확인하는 메뉴인지. |  |  |  |
```

규칙:

- 모든 식당에는 최소 3줄 추천 이유가 있어야 합니다.
- “맛있음”, “유명함”, “가성비 좋음”만으로는 부족합니다.
- 주요 메뉴마다 추천 사유를 적습니다.
- 코스요리/오마카세는 주요 코스와 접시별 특징을 확인합니다.
- 출처 근거와 AI의 부가 분석을 분리합니다.

## 샘플: 재슐랭 최근 3개월 가격대별 정리

재슐랭가이드는 제가 애정하는 맛집 유튜버 중 한 명을 기준으로 만든 샘플입니다.

예를 들어 다음처럼 요청할 수 있습니다.

```text
재슐랭 최근 3개월 맛집 리스트를 가격대별로 정리해줘.
가격무관, 10만원 이하, 5만원 이하, 3만원 이하 Top 10으로 나누고,
식당별 3줄 추천, 주요 메뉴별 추천 사유, 가격대, 위치, 출처를 같이 써줘.
```

실제로 아래 샘플 표를 만들 때 사용한 프롬프트는 이런 형태입니다.

```text
재슐랭 최근 3개월 맛집 리스트를 다시 정리해줘.

조건:
1. 가격무관 Top 10
2. 10만원 이하 Top 10
3. 5만원 이하 Top 10
4. 3만원 이하 Top 10

각 표에는 다음 컬럼을 포함해:
- 순위
- 식당
- 3줄 추천
- 메뉴별 추천 사유
- 가격대
- 위치
- 출처/주의

작성 규칙:
- 제목만 쓰지 말고, 왜 추천하는지 식당별로 최소 3줄 작성.
- “맛있음”, “유명함”, “가성비 좋음” 같은 피상적 표현 금지.
- 주요 메뉴마다 왜 먹어야 하는지 설명.
- 코스요리/오마카세라면 주요 코스와 세부 메뉴 특징을 써줘.
- 재슐랭 근거와 AI의 부가 분석을 구분해줘.
- Shorts는 제목만 믿지 말고 설명란, 해시태그, 가격, 위치까지 확인해줘.
```

그러면 `foodie-radar`는 단순 제목 목록이 아니라 다음처럼 정리합니다.

- 월간 순위, 반복 언급, 제목의 강한 추천 시그널을 분리합니다.
- 설명란/해시태그에서 식당명, 가격, 위치, 메뉴를 뽑습니다.
- 각 식당마다 최소 3줄 추천 이유를 씁니다.
- 주요 메뉴마다 왜 주문 후보인지 설명합니다.
- 코스요리/오마카세는 세부 코스가 확인되지 않으면 보류하거나 순위를 낮춥니다.

## 맛집러/인플루언서 리스트

현재 `creator-source-pool.md`에는 유튜브 공개 구독자수 기준으로 1차 맛집러/인플루언서 후보를 정리해 두었습니다.

다만 이 순위는 절대적인 맛집 신뢰도 순위가 아닙니다. 시작점에 가깝습니다.

사용자는 본인 취향에 맞게 맛집러를 편집할 수 있습니다.

- 1군으로 바로 추적할 맛집러
- 보조 레이어로 둘 맛집러
- 광고/협찬 리스크 때문에 보류할 맛집러
- 술집, 분식, 편의점, 프랜차이즈 중심이라 제거할 맛집러
- 파인다이닝이나 오마카세처럼 예산이 생기면 추적할 맛집러

예를 들어 저는 다음 기준을 중요하게 봅니다.

- 나와 비슷한 맛집 취향
- 나보다 훨씬 고급 맛집 취향
- 인생 맛집, 단골 맛집, 국내 1위, 분야 최고 같은 강한 추천 시그널
- 단순 홍보가 아니라 실제로 왜 가야 하는지 설명하는 능력

따라서 `foodie-radar`의 맛집러 리스트는 고정된 정답이 아니라, 각자의 취향에 맞게 계속 수정하는 레이더입니다.

## 재슐랭 Shorts 파훼 룰

재슐랭가이드처럼 Shorts를 많이 올리는 채널은 제목만 믿으면 누락이 생깁니다.

예를 들어 제목이 `진짜 비싼 곰탕집, 이거 괜찮은가요?`이면 식당명은 제목에 없고 설명란에만 있을 수 있습니다.

따라서 Shorts는 다음 항목을 모두 봅니다.

- 제목
- 설명란
- 해시태그
- 업로드일
- 자동 자막 또는 화면 텍스트
- 식당명 별칭과 지점명

## 사용 방법

1. 이 repo를 Obsidian vault 또는 GitHub repo로 사용합니다.
2. `restaurant-intelligence/` 아래 Markdown 파일을 매일 또는 주기적으로 업데이트합니다.
3. Codex나 다른 AI 도구에 `skills/restaurant-trust-intelligence/SKILL.md`를 스킬로 등록합니다.
4. 맛집 추천을 받을 때는 단순 검색이 아니라 출처, 신뢰도, 광고 리스크, 취향 적합성, 메뉴별 이유를 함께 요구합니다.
